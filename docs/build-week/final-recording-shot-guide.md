# Final Recording Shot Guide

Target length: **2:40–2:50**. Hard gate: **under 3:00**. Record only the public production site in a
clean browser window. Do not show terminals, cloud consoles, account menus, notifications, or raw
JSON.

## Shot list

| Time | Screen action | Visual proof | Narration purpose |

| 2:28–2:45 | Authority boundary | No Intune writes, OIDC, public allowlist, `store: false`, human judgment | Security close |

## Dynamic-value rule

Never narrate a stored snapshot ID, timestamp, count, setting, or finding from this repository.
Immediately before recording, read the current values from Mission Control and speak only what is
visible. If freshness is stale, a mode is unavailable, or the displayed device aggregate is not
understood, stop and use the [protected refresh runbook](../operations/demo-refresh.md).

## Privacy-safe framing

- Crop browser chrome if it shows an account, extension, profile, history, or unrelated tab.
- Do not open the Mission JSON, Actions logs, Cloudflare logs, Entra, Intune, or OpenAI Platform.
- Finding dialogs may show only the sanitized evidence IDs and pseudonyms already published.
- Avoid zooming so far out that human-review or reference-only qualifiers become unreadable.
- Use no background music unless TMCO Consulting, LLC owns or has documented usage rights.

## Capture settings

- 1920×1080 or 2560×1440, 30 fps, system cursor visible.
- Browser zoom 100–125%, choosing the highest readable setting that does not clip the dashboard;
  set operating-system scaling before recording.
- Clear narration, no silence longer than two seconds, and no animated transition longer than one
  second.
- Export H.264 MP4. Verify duration, audio, text legibility, and absence of private data before
  upload.
